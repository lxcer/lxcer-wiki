# 个人知识库

个人知识库（Personal Knowledge Base, PKB）是个体用于收集、组织、存储和检索个人知识、信息、想法和经验的系统。

## 核心理念

### 从消耗到积累

传统信息消费模式：
- 阅读 → 遗忘 → 再阅读 → 再遗忘

个人知识库模式：
- 阅读 → 提取 → 连接 → 积累 → 复利增长

### 知识的复利效应

> "知识的价值不在于存储，而在于连接。"

孤立的知识点价值有限，但当知识点之间形成网络，价值呈指数增长。

## 主要流派/方法

### Zettelkasten（卡片盒笔记法）

- **起源**: Niklas Luhmann
- **核心**: 原子化笔记 + 永久编号 + 连接导向
- **特点**: 笔记之间形成复杂的引用网络

### PARA方法

- **P**rojects（项目）: 当前正在进行的任务
- **A**reas（领域）: 长期责任范围
- **R**esources（资源）: 感兴趣的话题
- **A**rchives（归档）: 已完成或不活跃的项目

### 数字花园（Digital Garden）

- 公开、不断演化的笔记集合
- 强调"成长中的想法"而非"完成的观点"
- 接受不完美和持续迭代

### 第二大脑（Second Brain）

- Tiago Forte推广的概念
- 强调个人知识管理的四个阶段: 捕获 → 组织 → 提炼 → 表达
- CODE方法论

- **剪藏**: Obsidian Web Clipper, Notion Web Clipper, Raindrop
- **稍后读**: Pocket, Instapaper, Readwise
- **标注**: Readwise, Matter, Hypothesis
- **思维导图**: MindNode, XMind, Miro

## 实践原则

### 开始简单

1. **选择一个工具**并坚持使用（至少3个月）
2. **建立收集习惯**: 看到有价值的内容立即保存
3. **定期回顾**: 每周花时间整理和连接笔记

### 渐进式组织

- 不要一开始就追求完美分类
- 让结构自然涌现
- 定期重构和整理

### 连接优于收集

- 不要只是囤积信息
- 主动提取、总结、连接
- 用自己的话重新表达

### 输出驱动

- 知识库的价值通过输出来体现
- 写作、分享、教授是最好的学习方式
- 从消费模式转向生产模式

## 与LLM Wiki的关系

[[llm-wiki-pattern|LLM Wiki模式]]是个人知识库的一种高级实现：

- **自动化维护**: LLM承担传统PKM中最繁琐的簿记工作
- **规模化**: 可以处理远超个人能力的大量来源
- **深度连接**: LLM能发现人可能忽略的跨文档联系
- **持续更新**: 新来源自动触发相关页面的更新
