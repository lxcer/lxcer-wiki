# LLM Wiki模式

一种使用LLM增量构建和维护个人知识库的方法论，核心在于**知识的持久化积累**而非**实时检索**。

## 核心理念

### 与RAG的区别

| 维度 | RAG模式 | LLM Wiki模式 |
|------|---------|--------------|
| 知识存储 | 原始文档分块索引 | 结构化Wiki页面 |
| 知识获取 | 每次查询重新检索 | 已编译好的知识网络 |
| 积累性 | 无积累，每次从零开始 | 持续复合增长 |
| 合成能力 | 受限于上下文窗口 | 交叉引用已预建 |

### 关键洞察

> "维护知识库最繁琐的部分不是阅读或思考——而是 bookkeeping（簿记工作）。"

人类放弃Wiki是因为**维护负担增长快于价值**。LLM不会感到无聊，不会忘记更新交叉引用，可以一次处理15个文件。

## 三层架构

```
┌─────────────────────────────────────────┐
│           Schema 模式层                 │  ← CLAUDE.md
│    (配置规范，告诉LLM如何维护Wiki)        │
├─────────────────────────────────────────┤
│            Wiki 知识层                   │  ← wiki/
│    (LLM生成和维护的Markdown页面)          │
│    - 实体页面 (人、组织、产品)             │
│    - 概念页面 (理论、方法、框架)           │
│    - 来源摘要 (每份来源的总结)             │
│    - 综合分析 (比较、主题、演化)             │
├─────────────────────────────────────────┤
│           Raw 原始来源层                 │  ← raw/
│    (原始的、不可变的来源文档)              │
│    - 文章、论文、博客                      │
│    - 书籍笔记                             │
│    - 播客、视频笔记                       │
│    - 截图、图表                           │
└─────────────────────────────────────────┘
```

## 工作流程

### 1. Ingest（摄取）

```
添加来源 → LLM读取 → 讨论要点 → 创建/更新页面 → 更新索引
```

每处理一个来源，LLM会：
- 创建/更新**来源摘要页面**
- 更新相关的**实体页面**（提到的人、组织、产品）
- 更新相关的**概念页面**（理论、方法、框架）
- 更新**索引**和**日志**

### 2. Query（查询）

```
提出问题 → LLM搜索相关页面 → 合成答案 → （可选）归档为新页面
```

重要洞察：**好的答案可以被归档回Wiki作为新页面**。一次比较分析、一个发现的联系、一个问题的答案——这些都是有价值的，不应该消失在聊天记录中。

### 3. Lint（清理）

定期检查：
- **矛盾**：页面间的冲突声明
- **孤立页面**：没有入站链接的页面
- **缺失页面**：被提及但未创建的页面
- **过时声明**：被新来源取代的旧观点
- **缺失交叉引用**：应该有的链接

## 关键文件

| 文件 | 用途 |
|------|------|
| `CLAUDE.md` | 模式规范，告诉LLM如何维护Wiki |
| `wiki/index.md` | 内容导向的目录，所有页面列表 |
| `wiki/log.md` | 时间线日志，记录所有操作 |
| `raw/` | 原始来源文档（不可变） |

## 与Memex的关联

Vannevar Bush在1945年提出的Memex愿景：
- 个人化、策展化的知识存储
- 文档间的关联路径（associative trails）
- 强调连接本身的价值

LLM Wiki是Memex的现代实现，**解决了Bush未能解决的问题**——谁来做维护工作。

## 实践建议

1. **逐一摄取来源**：保持参与，阅读摘要，指导LLM强调重点
2. **使用Obsidian浏览**：图谱视图看连接，反向链接看引用
3. **定期Lint**：每周/每月检查矛盾、孤立页面
4. **归档好答案**：查询结果有价值时，保存为新页面
5. **迭代Schema**：随着使用，与LLM共同演化`CLAUDE.md`
